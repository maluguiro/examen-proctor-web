# ProctorEtic Frontend - Reglas para Codex

## Cómo trabajar
- Primero: dar un plan breve (máx 8 bullets) + lista de archivos a tocar.
- Después: aplicar cambios SOLO si se pidió explícitamente "aplicar".

## Restricciones (muy importante)
- NO refactorizar.
- NO mover ni borrar JSX/TSX.
- NO renombrar archivos o componentes.
- Cambios pequeños: máximo 3 archivos por tarea.
- Si el objetivo es UI/contraste: tocar SOLO className (Tailwind) o CSS puntual. No reescrituras.

## Validación obligatoria
- `npm run build` debe pasar.

## Entregables
- Diff mínimo.
- Pasos de prueba manual (2-3 pasos).
